# assets/

All raw game assets: sprites, music, and sound effects.
Nothing in this folder is imported by Godot directly — files are referenced
through `.import` metadata generated by the editor on first import.

All folders are currently empty; this document defines the structure and naming
conventions to follow as production begins.

---

## Folder Structure

```
assets/
├── music/              Background music tracks
├── sfx/                Sound effects
└── sprites/
    ├── characters/     Player character sprite sheets
    ├── enemies/        Enemy sprite sheets
    ├── environment/    Environmental objects and props
    │   └── items/      Collectables and pickups
    │   └── traps/      Traps
    └── weapons/        Weapon and projectile sprites
```

### Suggested folders (not yet created)

These will likely be needed as production progresses:

| Folder | Purpose |
|---|---|

| `sprites/vfx/` | Visual effects: hit sparks, dust clouds, explosions |

## sprites/characters/

One subfolder per character (e.g. `sprites/characters/Scavenger/`).

**Naming:** `{CharacterName}{Action}.png`

Both the character name and action are PascalCase, concatenated with no
separator — matching the CorgiEngine Rectangle convention.

| Example | Meaning |
|---|---|
| `ScavengerIdle.png` | Standing idle |
| `ScavengerWalk.png` | Walking |
| `ScavengerRun.png` | Running |
| `ScavengerJump.png` | In-air jump |
| `ScavengerDash.png` | Dash ability |
| `ScavengerCrouchIdle.png` | Crouch — stationary |
| `ScavengerCrouchWalk.png` | Crouch — moving |

Each file is a sprite sheet; frame layout is defined in the Godot
`SpriteFrames` resource, not encoded in the filename.

---

## sprites/enemies/

One subfolder per enemy type (e.g. `sprites/enemies/Crawler/`).

**Naming:** `{EnemyName}{Action}.png` — same PascalCase convention as
characters.

| Example | Meaning |
|---|---|
| `CrawlerIdle.png` | Standing idle |
| `CrawlerWalk.png` | Patrol walk |
| `CrawlerAttack.png` | Attack animation |
| `CrawlerDeath.png` | Death animation |

---

## sprites/environment/

Props, platforms, and background decoration that are placed as individual
nodes (not tiled).

**Naming:** `{object}_{variant}.png` — snake_case, with an optional variant
suffix for different states or materials.

| Example | Meaning |
|---|---|
| `crate_wood.png` | Wooden crate |
| `barrel_metal.png` | Metal barrel |
| `door_closed.png` | Door — closed state |
| `door_open.png` | Door — open state |

### sprites/environment/items/

Collectables and pickups.

**Naming:** `item_{name}.png`

| Example | Meaning |
|---|---|
| `item_health_small.png` | Small health pickup |
| `item_scrap.png` | Scrap currency pickup |

---

## sprites/weapons/

Weapon bodies and projectiles. Animated weapons follow the character/enemy
PascalCase pattern; static projectiles use snake_case.

**Animated (held weapons):** `{WeaponName}{Action}.png`

| Example | Meaning |
|---|---|
| `PistolIdle.png` | Pistol at rest |
| `PistolFire.png` | Pistol firing frame |

**Projectiles:** `projectile_{name}.png`

| Example | Meaning |
|---|---|
| `projectile_bullet.png` | Standard bullet |
| `projectile_rocket.png` | Rocket |

---

## music/

One file per track.

**Format:** `.ogg` (Godot streams `.ogg` without loading the full file into
memory — preferred for music).

**Naming:** `{area}_{mood}.ogg`

| Example | Meaning |
|---|---|
| `hub_calm.ogg` | Hub area — ambient/calm |
| `bar_intense.ogg` | Bar area — tense |
| `boss_fight.ogg` | Boss encounter |

---

## sfx/

Short one-shot audio clips.

**Format:** `.wav` (preferred for SFX; lower latency than `.ogg`).

**Naming:** `{category}_{action}.wav` — where category is the system or
object producing the sound.

| Example | Meaning |
|---|---|
| `player_jump.wav` | Player jumps |
| `player_land.wav` | Player lands |
| `player_dash.wav` | Dash ability fires |
| `enemy_hit.wav` | Enemy takes damage |
| `enemy_death.wav` | Enemy dies |
| `ui_select.wav` | Menu item selected |
| `pickup_scrap.wav` | Scrap collected |
| `checkpoint_activate.wav` | Checkpoint triggered |
