# assets/

All raw game assets: sprites, music, and sound effects.
Nothing in this folder is imported by Godot directly тАФ files are referenced
through `.import` metadata generated by the editor on first import.

All asset folders are currently empty; this document defines the structure and
naming conventions to follow as production begins.

---

## fonts/

Font files used by UI labels and debug overlays.

---

## music/

One file per track.

**Format:** `.ogg` (Godot streams `.ogg` without loading the full file into
memory тАФ preferred for music).

**Naming:** `{area}_{mood}.ogg`

| Example | Meaning |
|---|---|
| `bar_intense.ogg` | Bar area тАФ tense |
| `boss_fight.ogg` | Boss encounter |
| `hub_calm.ogg` | Hub area тАФ ambient/calm |

---

## sfx/

Short one-shot audio clips, organised by source category.

**Format:** `.wav` (preferred for SFX; lower latency than `.ogg`).

**Naming:** `{action}.wav` тАФ no category prefix needed since the subfolder
already encodes the source.

### sfx/characters/

Sounds produced by the player character.

| Example | Meaning |
|---|---|
| `dash.wav` | Dash ability fires |
| `jump.wav` | Player jumps |
| `land.wav` | Player lands |

### sfx/enemies/

Sounds produced by enemies.

| Example | Meaning |
|---|---|
| `death.wav` | Enemy dies |
| `hit.wav` | Enemy takes damage |

### sfx/environment/

Sounds from interactive world objects.

#### sfx/environment/items/

| Example | Meaning |
|---|---|
| `pickup.wav` | Generic item pickup |
| `scrap.wav` | Scrap collected |

#### sfx/environment/traps/

| Example | Meaning |
|---|---|
| `activate.wav` | Trap activates |

### sfx/ui/

| Example | Meaning |
|---|---|
| `checkpoint_activate.wav` | Checkpoint triggered |
| `select.wav` | Menu item selected |

### sfx/weapons/

| Example | Meaning |
|---|---|
| `pistol_fire.wav` | Weapon fires |
| `pistol_reload.wav` | Weapon reloads |
| `revolver_fire.wav` | Weapon fires |
| `revolver_reload.wav` | Weapon reloads |

## sprites/characters/

One subfolder per character (e.g. `sprites/characters/scavenger/`).

**Naming:** `{action_name}.png`

| Example | Meaning |
|---|---|
| `crouch_idle.png` | Crouch тАФ stationary |
| `crouch_walk.png` | Crouch тАФ moving |
| `dash.png` | Dash ability |
| `idle.png` | Standing idle |
| `jump.png` | In-air jump |
| `run.png` | Running |
| `walk.png` | Walking |

Each file is a sprite sheet; frame layout is defined in the Godot
`SpriteFrames` resource, not encoded in the filename.

---

## sprites/enemies/

One subfolder per enemy type (e.g. `sprites/enemies/crawler/`).

**Naming:** `{action_name}.png`

| Example | Meaning |
|---|---|
| `attack.png` | Attack animation |
| `death.png` | Death animation |
| `idle.png` | Standing idle |
| `walk.png` | Patrol walk |

---

## sprites/environment/

Props, platforms, and background decoration placed as individual nodes
(not tiled).

**Naming:** `{object}_{variant}.png` тАФ snake_case, optional variant suffix for
different states or materials.

| Example | Meaning |
|---|---|
| `barrel_metal.png` | Metal barrel |
| `crate_wood.png` | Wooden crate |
| `door_closed.png` | Door тАФ closed state |
| `door_open.png` | Door тАФ open state |

### sprites/environment/items/

Collectables and pickups.

**Naming:** `{name}.png`

| Example | Meaning |
|---|---|
| `health_small.png` | Small health pickup |
| `scrap.png` | Scrap currency pickup |

### sprites/environment/traps/

**Naming:** `{name}_{state}.png`

| Example | Meaning |
|---|---|
| `spike_idle.png` | Spike trap at rest |
| `spike_active.png` | Spike trap extended |

---

## sprites/tiles/

Tileset source images used to build Godot `TileSet` resources for terrain and
platforms.

**Naming:** `{tileset_name}.png` тАФ one image per tileset.

| Example | Meaning |
|---|---|
| `metal_industrial.png` | Industrial metal tileset |
| `scrapyard.png` | Scrapyard tileset |

---

## sprites/ui/

HUD elements, icons, and menu art.

**Naming:** `{element}_{state}.png`

| Example | Meaning |
|---|---|
| `button_idle.png` | Button тАФ default state |
| `button_pressed.png` | Button тАФ pressed state |
| `healthbar_fill.png` | Health bar fill |
| `sanitybar_fill.png` | Sanity bar fill |

---

### sprites/vfx/enemies/

| Example | Meaning |
|---|---|

### sprites/vfx/weapons/
